函数式编程

函数式编程的思想

面向对象的思想需要关注用什么对象完成什么事情，而函数式编程思想类似于数学里面的函数，他关注的式对数据做了什么操作。不关心谁调用的，不关心方法名字，只关心对数据的操作。

lambda 表达式

函数式接口：如果一个接口只包含唯一一个抽象方法(JDK1.8之后可以包含非抽象方法，这里只看抽象方法)，那么他就是一个函数式接口。

最初：一个接口，一个实现类，只使用一次。

由于这个类不会单独使用，所以可以将其声明为一个成员内部类，但是又有可能需要定义静态的方法和 成员所以，将类声明为一个静态内部类。

还可以进一步简化就变成局部内部类，直接将类定义在方法体中：

采用匿名内部类进一步简化，匿名内部类对象的类型是接口或者父类，借助接口或者父类来实例化对象，new 接口/父类，当为接口时需要重写接口方法。

最后根据lamda进行简化，由于对象的类型是接口或者父类，当为接口时，根据函数式接口的定义，使用lamda表达式时，要求接口为函数式接口，只保留匿名内部类的（） {函数体}，并用-> 进行指向。

Stream 流

这是函数式编程中最常用的操作，它可以操作集合和数组

准备

快速开始

  • 需求要找到年龄小于18 的作家的名字且去重。

  • 传统写法：

传统写法很难避免大量的 if语句和for语句的嵌套 ，导致代码可读性降低。

  • Stream流写法

Stream 流可以用来操作数组和集合，可以将数组和集合先变成一个个流数据便于我们的处理，变成流数据也只需要通过一个stream方法就能够实现。

常用操作

创建操作

上面我们说过Stream流是用于操作数据和集合的，所以对数组和集合都有各自的创建方式。

  • 单列集合： 集合对象.stream()
  • 数组 Arrays.stream()；Stream.of()
  • 双列集合 Map，要先转成单列集合再通过stream方法转成流

中间操作

将数组和集合转成流之后，我们需要对流中的数据进行一些操作，以达到我们的目的。

  • filter

过滤，将满足条件的流数据继续保留在流中。

实际上就是每一个流数据都会经过filter 中的实现方法，进行筛选。

  • map

可以对我们流数据中的内容的类型进行转换，转换后后面接着的流方法获取到的就都是转换后的类型了。

可以通过debug 查看 流式连每次发生的变化

  • distinct

去重操作，注意：这里的底层使用的是Object 的equals 方法进行判断的，所以相当于使用的是 “==” 进行判断，只有当地址相同才算相同，所以当我们要操作自己的类的时候要重写equals方法。

  • sorted

排序

  • limit

设置流的长度，比如说你现在流中有3 个元素，你只希望有两个就可以通过limit指定要几个元素。

  • skip

跳过前n个流元素

  • flatMap

终结操作

在使用Stream流的时候，必须存在终结操作，不然所有中间操作都不会发生。

  • forEach

遍历所有stream 的内容。

  • count

计算所有stream的个数。

  • max / min

取出流数据中的最大/最小值，得到结果是一个optional的类型，可以用 get方法获取里面的内容。

  • collect

把当前流数据转换成一个集合（list，set，map）。

    • list
    • set和list一样,使用toset就行了
    • map，需要指定怎么转成key，怎么转成value，所以toMap有两个参数，第一个参数是Key指定的规则，第二个参数是value指定的规则。两个参数都需要用匿名内部类或者lambda表达式来完成

需求：author 转成 map，姓名做key，书的集合做value

  • anyMatch

只要流数据中有任意一个满足的就会返回true。

  • allMatch

只有流数据中所有都满足给定条件才会返回true。

  • noneMatch

所有都不符合条件 返回true

  • reduce 归并

对流数据按照指定的计算方式计算出一个结果。

Optional

我们在写一些查询语句的时候，拿到查询的结果可能是空的，如果我们还要调用对象的方法就会出现空指针异常，所以我们会加上 if 去判断是否为空，但是如果对象多起来的话，if就会很多显得很乱，而且有些时候还可能会忘记加上。

创建对象

​ Optional就好像是包装类，可以把我们的具体数据封装Optional对象内部。然后我们去使用Optional中封装好的方法操作封装进去的数据就可以非常优雅的避免空指针异常。

​ 我们一般使用Optional的静态方法ofNullable来把数据封装成一个Optional对象。无论传入的参数是否为null都不会出现问题。

而且在实际开发中我们的数据很多是从数据库获取的。Mybatis从3.5版本可以也已经支持Optional了。我们可以直接把dao方法的返回值类型定义成Optional类型，MyBastis会自己把数据封装成Optional对象返回。封装的过程也不需要我们自己操作。

安全消费值

​ 我们获取到一个Optional对象后肯定需要对其中的数据进行使用。这时候我们可以使用其ifPresent方法对来消费其中的值。

​ 这个方法会判断其内封装的数据是否为空，不为空时才会执行具体的消费代码。这样使用起来就更加安全了。

获取值

​ 如果我们想获取值自己进行处理可以使用get方法获取，但是不推荐。因为当Optional内部的数据为空的时候会出现异常。

安全获取值

​ 如果我们期望安全的获取值。我们不推荐使用get方法，而是使用Optional提供的以下方法。

  • orElseGet

    获取数据并且设置数据为空时的默认值。如果数据不为空就能获取到该数据。如果为空则根据你传入的参数来创建对象作为默认值返回。

  • orElseThrow

    获取数据，如果数据不为空就能获取到该数据。如果为空则根据你传入的参数来创建异常抛出。

过滤

​ 我们可以使用filter方法对数据进行过滤。如果原本是有数据的，但是不符合判断，也会变成一个无数据的Optional对象。
