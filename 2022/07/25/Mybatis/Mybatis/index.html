环境配置

environments

  • < environments default=””> < /environments> ，中间存放多个环境，不同环境通过不同的id进行 标识，切换环境时，修改默认值。

  • < transactionManager> 事务管理器，有两个选择JDBC和MANAGED 。

  • < **dataSource**> 连接数据库，有三种内建的数据源类型（也就是 type=”[UNPOOLED|POOLED|JNDI]”）：

    • UNPOOLED：没有连接池
    • POOLED：有连接池（默认）

properties

除了直接指定，还可以通过引入资源文件，db.properties 使用 < properties> 标签进行读取。

注意：< properties>标签一定要写在environments之前configuration之后。

typeAliases

类型别名可为 Java 类型设置一个缩写名字。 它仅用于 XML 配置，意在降低冗余的全限定类名书写

多对一处理

例子

老师和学生的关系中，

学生—>老师： 多个学生 关联 一个老师 【多对一】

老师—->学生： 一个老师 管理（集合） 多个学生 【一对多】

创建实体类及其接口，此时会有两个接口，如果都放在Dao下，不同容易管理，所以可以放在resources下，注意：一定要建立和mapper同样的包的路径，否则容易找不到。

需求：查询所有学生以及对应老师的信息

问题：teacher列为null

原因：查询得到的是s.id，s.name，t.name，映射到Student类上，前两个是对应的，但是第三个属性，Student中的是Teacher 对象，所以无法映射，实际上还是字段名和属性名不一致的问题。

  • 方法一：子查询（根据查询进行嵌套处理）

    修改sql语句，先查询所有的学生，然后根据学生的tid去查询老师。所以需要两个查询完成。又因为返回的是Student类，Student类中有Teacher类对象，所以还需要对最后的结果做结果集映射。而Student前两个字段和属性名是相同的，而第三个属性是类的对象在映射是有一些区别。

​

  • 方法二：连表查询（根据结果进行嵌套处理）

​ sql可以不变，但是这里为凸显连表和提高效率，使用left join来完成，这种做法的主要思路直接指定映射类型，并且在指定类型后，对这个类型中的值再次进行映射。

动态SQL

概念：根据不同的条件生成不同的sql语句，有点类似根据 if 判断是否满足什么条件，然后对sql语句字符串进行拼接。

< if>标签

< where> 标签

有些时候，我们在写动态sql时，第一个拼接条件不符合时，就可能会出现 **XXXXX where and XXXX ** 的情况，这样的语句是不合规的。所以我们习惯性用 < where > 标签来代替在主句上写where

< when>标签

类似于java中的switch，多个条件满足时，只会匹配一个条件。

< set> 标签

和 where标签一样，用于update语句中，他会除去最后一个 逗号。

sql片段

一个xml文件中肯定存在很多被重复写的sql语句，可以使用sql片段对大量重复的代码进行截取，并复用。
