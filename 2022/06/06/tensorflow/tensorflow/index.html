tensorflow

创建张量

tensorflow中所有数据甚至一维的数据也可以理解成一个tensor,创建一个张量用到的时tf中的constant函数。

tensorflow中的数据类型

属性

tensor表示tf的对象

tensor.device：查看当前创建的tensor是在cup还时gpu化境，tensor.cup(),tensor.gpu()：用于切换

tensor.numpy() ，tensor转为np.array类型

tensor.ndim 数据的维度

tf.is_tensor(tensor) 判断是否为tensor类型

tensor.dtype 返回对象的类型

tf.conert_to_tensor(), tf.cast() numpy转为tensor

bool<–>int

tf.Variable(tensor) 将tensor转为一个可以被训练的参数，转换后的对象可以具有一些自动求导的功能，自动记录梯度的信息。但是它本身还是一个tensor

创建tensor

用的的较多的类型时 int32和float32

tf.convert_to_tensor(data)

传进去的参数是数据而不是shape

tf.zeros(shape)、tf.ones(shape)

用0/1填充，参数是shape不是数据

tf.zeros_like(tensor)、tf.ones_like(tensor)

创建和传入tensor相同维度的全用0/1填充的tensor对象。

tf.fill(shape,data)

第一个参数传入维度，第二个参数表示用什么数据填充

tf.random.normal(shape,mean,stddev)

第一个参数传入维度，第二个参数指定均值（默认0），第三个参数指定方差（默认1）。用正态分布来算计初始化权重。

索引切片

start“：” end 选择[start，end)的元素。start默认0开头，end默认-1结尾。

start : end ：step 每隔2取一个

tf.gather(tensor,axis,indices=[])

提供无规律采样，第一个参数为带采样的tensor对象，假设为[4,28,8]的tensor对象，axis表示取那个维度，indices是一个元组，提供要采的下标、假设这个tensor表示的意思是有4个班级，每个班级28个学生，有8门课程。

tf.gather(tensor,indices=[])

传入元组，元组的下标和但操作的tensor对象对应，参数元组的值表示对应下标的值

维度变换

reshape

不改变源数据。tensorflow默认的存储格式为[b，h，w，c]

transpose (tensor,perm=[])

会改变源数据，perm默认不填时将数据整个转置。

tf.expand_dims(tensor,axis)

添加维度，axis表示在哪添加。

tf.squeeze(tensor,aixis)

删除某个维度上值为1的维度，axis默认删除所有值为一的维度

Broadcasting

一个tensor的维度称最右端的为小维度，最左端的是大维度，当一个维度为[784,10]和一个维度为[10]的两个tensor相加时，会先使得小维度对其，然后向高纬度扩张，先由[10]–>[1,10]，在和另一个tensor对齐有[1,10]—>[784,10]在进行运算。Broadcasting是一个运算时，优化的手段看似扩张了数据的维度但是实际的数据并没有发生改变，

若低纬度的不相同不能进行broadcasting，broadcasting只关注维度上不为1的取值的是否相等，为1表示可扩张称和另一个tensor在该维度上相同的值。[4,32,14,14]和[1,32,1,1]，虽然最低两个就不相同，但是1表示可扩张，所以[1,32,1,1]—>[4,32,14,14]是可以进行broadcasting的。由于这是一种优化手段所以在使用时，可以不需要使用tf.broadcast_to函数，直接进行tensor之间的运算的时候，编译器自动检查。

tf.tile(tensor,[])

和Broadcasting作用一样用于扩张维度，但是用tile得到的结果所占用的内存比用broadcasting得到的结果大。

元组[a,b]根据元组的下标表示对tensor对应下标扩大a,b倍

合并于分割

合并

tf.concat([tensor1,tensor2],axis=)

两个tensor对象，在axis维度上进行合并,除axis维度其他都需要相等

tf.stack([tensor1,tensor2],axis=)

在tensor1的axis维度上堆叠tensor2 会增加一个维度所有维度都需要相等

分割

tf.unstack(tensor,axis=)

stack的逆操作，axis上有几个，就会被拆分为几个tensor

tf.split(tensor,axis,num_or_size_splits)

和unstack相比split可以指定分割个数，num_or_size_splits可以是数，也可以是一个元组

数据统计

范数

一范数

所有元素绝对值的和。

二范数

所有元素平凡和开根号。

无穷范数

所有元素的绝对值的最大值。

max/min/mean

argmax/argmin

最大最小值的位置

比较

排序

Top_k

返回前k个，indices取得索引，values取得值

高阶操作

tf.where(tensor)

返回一个[n,tensor.ndim]的tensor,n表示tensor中包含的值为True的个数。tensor.ndim表示元素的所在位置。可以结合tf.gather_nd取得原来tensor中值为True的结果

tf.where(cond,A,B)

cond是一个值为True/False的tensor，where可以从A中选择值为cond值为True的值，B中选择cond值为False的值。

数据集处理

基本流程

1.加载数据集

2.对y进行转为int64，one_hot编码

3.对x将x处理为float32类型，0 ~ 1/ -1 ~ 1的范围

4.把x转为tensor对象

5.x,y进行相对位置的映射

6.打乱数据，设置batch

tf.split(tensor,num_or_size_splits=[])

全连接层

net = tf.keras.layer.Dense(out_shape)

传入输出的维数，返回一个全连接层对象，调用返回对象时，传入输入数据tensorflow会自动完成w*x+b返回输出结果。net.kernel是网络的权重的参数，net.bias是网络的偏执的参数

model = tf.keras.Sequential([Dense1,..])

创建多层的模型，传入每一层输出的维数，构建模型后，调用model传入输入数据，tensorflow会自动完成数据依次每层的流动计算。

计算梯度

with tf.GradientTape() as tape:

把需要跟踪的梯度信息放在里面，tensorflow会自动跟踪梯度变化，只有被包含在这个区域范围的loss函数才会被跟踪，在区域外部的变量不会被跟踪。

tape.watch([])

传入待跟踪的参数，如果不调用这个函数的话，那么参数的类型就应该改为tf.Variable类型。

tape.gradient(loss,[])

loss为在区域内定义的变量，[]为在区域内传入tape.watch中的需要跟踪的参数，tensorflow会自动求导计算。调用之后会释放gradien的内存及相关信息

tf.keras

datasets

自带的训练数据集合包括Mnist，用load_data可以进行下载。

Metrics

存放一次训练和下一次训练过程中间的loss、acc等一些值

Compile、Fit、Evaluate、Predict

定义好网络之后对于数据集的训练、梯度计算、损失函数、循环次数、测试环节在一般的网络算法中都是比较固定的。Keras就提供了比较方便的简写的API。

定义网路

Compile(optimizer,loss,metircs)

传入优化器、损失函数、评估方式

Fit(data,epoch,validation_data,validation_freq)

训练集，迭代次数，验证集，迭代过程中进行测试的频率。可提前终止

Evaluate(data_test)

训练结束后进行测试，fit是便训练边测试。

Predict

自定义网络

OOD方式构建自己的网络。自定义网络必须继承keras.layers.Layer 或 keras.Model

keras.layers.Layer

需要重写call方法，不能使用tf.constant来创建tensor，因为层之间可能不仅仅只是简单的孤立的一个tensor，可能需要完成层之间的组织，所以应该用自带的函数add_variable()来创建tensor参数

keras.Model

创建的是一个网络，同样也需要重写call。相比于layer，Model还提供实现Compile、fit、evaluate、predict的接口。

模型保存和加载

model.save_weights(path=””)

给定路径就可以保存训练时的权重参数。轻量级保存，不保存模型结构。

model.load_weights(path=””)

给定路径取得模型存放位置。但是在加载参数之前需要先创建模型对应网络的对象。

model.save(path=””)

完全保存，包括模型的结构。效率低

tf.keras.models.load_model(path=””)

和model.save配套的加载模型的函数。不需要重新创建网络对象。
