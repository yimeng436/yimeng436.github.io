23. 合并 K 个升序链表 - 力扣（LeetCode）

方法一：小根堆

题目说了每个链表都是排好序的，所以第一次选择的元素肯定是所有链表中某一个的头元素，
然后这个头元素的指针就往下一个偏移，list = list.Next，在进行下面和其他头元素的对比，
相当于还是从所有指针指向的头元素中，选择最小的一个，每次要获取最小值，这很容易就想到了堆这个数据结构。

首先先把所有元素的头节点加入到堆中，然后循环获取堆顶，如果堆顶元素的Next 不为空，就把堆顶的Next也加入到堆中
依次获取堆顶元素，直到堆空。

方法二：分治

合并两个有序链表非常的简单，那合并K个链表，不就可以依次两两合并，得到最终结果。这里的两两合并可以通过
分治方式来实现，每次都合并 len(lists )/2 的list

O(n)的时间复杂度，就只能遍历一遍数组，应该要能够想到空间换时间的思想，用到Map来辅助完成。

方法三

上面可以算是中等的题目了，如果面试官还要深挖就只能在空间复杂度上了，上面用到了 map 空间复杂度是 O(n)，

如果要再难一点的话就是要求常数的空间复杂度。

要求常数空间复杂度就需要用到另一个思想，原地哈希。

把数组当作hash表，自己设计一个映射规则。这题中，我们定义的映射规则是，把值为i映射到下标为i-1的位置上
这样在最后遍历的时候，只要i-i的位置上值不为i就是第一个未出现的正数。

例子：

3, 4, -1, 1

0, 1, 2, 3

第一次遍历到3的时候，判断是不是大于0，小于len(nums)，是则和nums[i]-1为索引的值互换

得到如下结果：

-1, 4, 3, 1

0, 1, 2, 3

继续判断当前的nums[i]是不是大于0，小于len(nums)，是则和nums[i]-1为索引的值互换

第一次和3交换后是-1,所以遍历下一个索引，i++



-1, 4, 3, 1

0, 1, 2, 3

nums[i]=4,和上面一样的判断和交换方式得到：

-1, 1, 3, 4

0, 1, 2, 3

交换后 nums[i]，i=1的值为1 还是满足大于0，小于len(nums)，所以继续和索引为nums[i]-1的值互换得到

1, -1, 3, 4

0, 1, 2, 3

同样现在i=1的位置是-1了，i++，后面的位置都放好了其实，但是如果只是按照上面的判断逻辑的话会出现死循环，
还需要加上nums[nums[i]-1]!=nums[i] ，一是为了防止死循环，而是可以做到去重。

按照顺序前往后放 数字和操作符，然后再从后往前计算结果，这样是错的。比如说下面这个测试用例

为了更加通用，我们把括号和更高优先级的运算符也考虑进去。

对于括号：

  • “(“ 左括号直接入运算符的栈

  • “)” 右括号，一直计算括号内的值，直到遇到左括号

对于运算符： 由于只知道当前的运算符，没办法直到后面的运算符的优先级，所以就和前面压入的运算符的优先级作比较

如果前面的优先级高，就可以直接去计算前面的值，直到遇到优先级比当前低的运算符，把当前运算符压入栈中

对于数字：一直取数字，直到下一位不是0-9，前面的整个是当前操作数

和为K的子数组

560. 和为 K 的子数组 - 力扣（LeetCode）

前缀和本身很简单，就是去记录当前下标之前的所有的元素之和。如果题目出现了连续子数组求和，求xx，这样的字眼可以考虑一下前缀和

对于这题，比较容易想到暴力解，时间复杂度 O(n2)。

这题要求连续子数组和 为 K，也就是要找到数组 nums下标 i到j，使得和为k。

再结合前缀和，我们有了下标为i的前缀和 preSum_i 以及下标为j的前缀和 preSum_j，要求 i到j的和不就是 preSum_j-preSum_i
。那么我们就只要，每次出现了preSum_j-preSum_i ==k，把他的和累加起来就行了，存储的化就直接用map去存储就好了。时间复杂度O(n)。

但是当遍历到 j的时候，我们没办法知道哪个i和当前j，计算preSum_j-preSum_i ==k，所以我们移项，k是固定的，我们前面又统计了 j 以前的所有前缀和，只要 preSum_j-k ==preSum_i，就表示前面出现了 满足preSum_j-preSum_i ==k这个式子的 前缀和 i。

找到两个数组的前缀公共数组

2657. 找到两个数组的前缀公共数组

通常首先想到的就是，数组+两个map去实现，map记录前面出现过的数字，遍历当前数字的时候，去判断另一个数组有没出现过。

最巧妙的算法是利用位运算实现，首先题目说到了n的排列，所以数组的元素是不重复的。所以我们可以用一个uint去代替暴力法的两个map。

比如说对于数组 [1,3,2,4]，用一个uint变量去统计数组里面有哪些数。

题目要求公共前缀，也就是在下标 i 下，要计算出两个数组的交集，而上面这种方式我们可以统计出两个下标为 i 的时候，两个数组前面分别包含了哪些数字，然后两个 uint 的变量做交集，计算出交集结果1的位数就能够得到结果。
