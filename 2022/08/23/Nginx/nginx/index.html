Nginx

什么是Nginx

​ Nginx是一个高新能的Http反向代理web服务器。在访问量上来之后，我们会用更多的去处理我们的请求，但是由于Session又是不共享的，所以可能导致我们现在访问到的是服务器1，等一下访问到了服务器2，但是服务器2说我没有登录等一些问题，所以我们希望能够帮我们控制转发请求的中间服务器。并且最好这个服务器能够还有一些别的功能如：负载均衡等。这个中间服务器就是Nginx。

反向代理

在讲反向代理之前，我们先说一下什么是正向代理。

正向代理：我么在国内如果要访问 www.google.com ，我们直接访问肯定是不能访问的。我们需要经过一个代理服务器，让代理服务器去访问这个 www.google.com，这个过程就叫做正向代理。 这个代理服务器是需要我们自己去配置的。

反向代理：首先反向代理不需要浏览器进行任何配置，它是一个网址对外暴露的一个服务器，假设 www.google.com 对外暴露的端口号为 9000的服务器，客户其实不知道这是一个反向代理服务器，客户访问这个端口号以为访问的就是目的的网址的真实服务器的ip，但实际上访问客户访问9000时，反向代理服务器会将请求进行转发到正真的网站的服务器上，但是这个转发的过程客户是不可知的。

动静分离

为了加快网站解析的速度，我们可以把动态页面和静态页面由不同的服务器来解析，降低单个服务器的压力，实际上还是有点类似于负载均衡的思想。

动静分析需要有一个专门的服务器存放静态资源，Nginx只需要在这个服务器中直接拿静态资源数据即可，而动态资源还是由tomcat解析。

要实现负载均衡，只需要再nginx的配置文件中的http{ } 中 使用：

upstream + 自定义一个服务器列表的名字{ server port; server port }

在location 配置中讲 自定义服务器列表配置进去

  • 先找到 http{} 部分

  • 在里面进行添加 upstream xxx

  • 将我们定义的服务器列表配置到location

nginx 默认是轮询访问 实现负载均衡。

还有一些其他的分配方式：

  • 权重

在自定义服务列表配置ip的后面，加上 weight=xxx，就能够指定某个ip的权重，权重越大分配的请求也就越多。

  • ip_hash

用户最开始访问到的是哪个服务器后面就只能访问到这个服务器，在自定义服务列表中加上ip_hash实现。

  • fair

根据服务器响应时间来分配，谁短给谁。

Nginx原理

实际上在linux启动nginx服务器之后，会有两种进程 一个是 maskter 一个是 worker，其中worker可有多个。

用户发起请求到master，master会将信号发送给地下的worker，每个worker以争抢的方式得到一个任务，再利用反向代理找到tomcat完成任务返回。

一个master配多个worker的好处在于每个worker是一个独立的进程，不需要加锁，一个进程出问题了不会影响整个服务器的允许，能够实现不重启服务器而实现配置重新加载。
