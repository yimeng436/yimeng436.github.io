HttpServlet、GenericServlet，所以只需要在自己的普通类上继承这个类。

Servlet原理

先根据浏览器的请求路径（这里是/hahaha），会先在web.xml 进行查找，是否有这么一个映射路径，有这拿到他的<servlet - name> ，再从注册的servlet中查找是否注册了这个servlet，如果注册了就会找到注册的这个类，就会访问到这个项目。

访问这个项目的时候会携带两个参数，请求和响应，tomcat服务器会去调用servlet的Service方法，并且把两个参数传递给Service方法，Service方法是在HttpServlet类中实现的，所以会执行HttpServlet中的Service方法，这个Service方法会根据请求这个参数获取请求方式，然后执行对应的方法，但是由于我们自己的类重写了doGet和doPost方法，所以就会执行我们编写的代码，执行结束后，将响应按照原路返回给tomcat服务器，tomcat会读取响应信息，并会将响应到浏览器上。

ServletContext

web容器在启动的时候，他会为每一个web程序都创建一个对应的ServletContext对象，他表示当前的web应用，并且管理整个web应用，这个ServletContext对象是一个全局唯一的对象，不同的servlet可以通过这个对象进行保存一些数据和内容。

  • 实现数据共享

    在一个servlet中，通过ServletContext对象的setAttribute（k,v）方法以键值对的形式添加内容。

    再通过另一个servlet，通过ServletContext对象的getAttribute（k）就可以获得其他servlet在ServletContext中保存的值。

  • 实现请求转发

    访问一个页面时，通过ServletContext对象的getRequestDispatcher（String url），获得转发的对象，再由他调用forward方法就可以实现请求转发。

注：使用RequestDispatcher，浏览器上的请求地址不会改变，还是输入的地址。有点类似于DNS的递归查询。如A要请求一个资源，但是这个资源在C这，请求转发的做法就是，A—>B，B—>C， C—>B，B—>A。

  • 读取资源文件

    首先，要读取文件肯定需要知道文件的地址，传统IO方法，可以直接通过文件的绝对地址来获得，但是在web应用中，由于要传给web服务器，所以绝对地址不太好找。

    每次启动一个web应用程序都会生成对应的target 文件夹，这个就是表示web应用的目标文件，然后所有的主程序中的main下面的java文件夹和resource文件夹下的所有内容都会被保存在 /项目名/WEB-INF/classes文件夹下，这个也被称为类文件夹。

    那既然所有mian下的文件都会被保存在web应用的目标文件下的类文件夹下，而’‘/’‘ 可以表示当前的web应用，所以在读取文件时，就可以通过**/WEB-INF/classes/对应资源文件的名字**。

用于给客户端响应信息。

简单的分类

向浏览器发送数据的方法

向浏览器发送响应头的方法

使用

  • 向浏览器输出消息

  • 下载文件

    ​ 做法：

    ​ 1.获取下载文件的路径

    ​ 2.让浏览器支持下载我们需要的东西

    ​ 3.获取下载文件的输入流

    ​ 4.创建缓冲区

    ​ 5.获取OutputStream对象

    ​ 6.（假设是下载的东西是文件）将FileOutputStream流写入到缓冲区

    ​ 7.使用OutputStream将缓冲区的数据输出到客户端

  • 实现重定向(注意对比请求转发)

    相比与请求转发，重定向的做法类似于DNS的迭代查询，如果A要请求的资源在C服务器，但是A请求了B，B知道要去请求C，所以B告诉A,C的地址，A在重新去访问C，这种做法会改变请求路径。

    可以看到请求redirect的状态码是302，表示的就是重定向

    • 常见应用：用户登录

      做法：启动tomcat时，进入index界面，提示需要输入用户名密码，提交表单后进入，需要对请求进行处理，处理后重定向至success.jsp界面。

    对比请求转发：

    • 相同点：

      页面都会跳转

    • 不同点：

      请求转发：Url不会发生变化

      重定向：Url会发生变化

Cookie、Session

两个都是用于保存会话的技术，从打开浏览器到关闭浏览器之前（是浏览器不是其中一个页面），这一整段的时间都是一个会话。

网站如何证明被用户访问过

  • 服务器给客户端一个标记，客户端下次访问时会带上这个标记；Cookie
  • 服务器对用户的访问进行登记，有访问请求是就会进行匹配；Session

Cookie

这是一个客户端技术

但是这个时候Cookie的存活时间只有在会话期间，如果将浏览器关闭在重启，第一次就还是不能拿到设置的Cookie的值。 可以通过设置cookie的MaxAge来设置存活时间。

Cookie的特点：

  • 一个浏览器的Cookie的数量有上线
  • 一个站点的Cookie的数量有上限
  • 一个Cookie的大小有上限4kb
  • 一个Cookie只能保存一个信息

删除Cookie：

  • 不设置MaxAge
  • 在另一个Servlet中将Cookie的MaxAge设为0

Session

这是一个服务器端技术，可以保存用户的会话信息。

  • 服务器会给每一个用户（浏览器）创建一个Session对象，只要浏览器没有关闭Session就会一直存在。（比如在登录B站时，登录之后，用户的状态一直会处于以登录的状态，而不需要切换一个页面就需要重新登录一次）

session注销

Session、Cooike对比

  • Cookie的数据是保存的客户端的，Session的数据是保存在服务器端的
  • Session对象由服务器创建
  • Session可以保存Object，Cookie只能保存String

JSP

JSP: java server pages。Java服务器端页面，和Servler一样，用于开发动态web技术。

其特点：

  • 语法类似html
  • 可以嵌入java代码
  • 和HTML相比
    • HTML 只提供静态数据
    • JSP嵌入Java代码可以提供动态数据
