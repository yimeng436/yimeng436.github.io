Go

Helloworld

首先一个go的程序需要有一个main 包，main包中包含一个main函数，只有main包会被编译成可执行文件。

通过go build xxx.go 指令可以将go文件编译成二进制可执行文件，再执行这个exe文件

go run xxx.go 可以编译且执行go文件

  • 在编译时，编译器会将程序运行依赖的库文件包含在可执行文件中，所以，可执行文件
    变大了很多。

  • 如果我们先编译生成了可执行女件，那么我们可以将该可执行文件拷贝到没有go开发环
    境的机器上，仍然可以运行
  • 如果我们是直接go run go源代码，那么如果要在另外一个机器上这么运行，也需要go
    开发环境，否则无法执行。
  • go run运行时间明显要比第一种方式 长一点点

函数

包

一个包下创建多个go文件，要想其他文件能够使用这个包下的函数的话，函数的首字母需要大写，因为大写相当于是 public 小写相当于 private。并且同除了同一个文件下不能有同名函数以外，同一个包下也不能有同名函数。

init函数

一个源文件下可以包含一个init函数，全局变量、init函数、main函数的执行顺序是 全局变量 -> init函数 ->main函数

defer

类似于 Java 的finally语句，一般用于释放资源，他会在函数返回结束之后一刻才会被调用，defer 后面可以跟函数 也可以跟语句，多个defer 被同时调用时会逆序执行，类似于栈。 并且他会将defer中的值一起压入栈。

主要用于关闭资源，有点像Java的finally。

闭包

闭包也称为匿名函数， 闭包可以赋值给一个变量，通过变量来调用函数，也可以定义完之后直接调用。

也可以将函数作为返回值。

可以发现，将方法赋值给一个变量之后，方法内部定义的局部变量的数据发生的改变会保留，也就是说函数中的变量只会被初始化一次，不会每次调用都初始化。所以闭包在准确一点的定义是 在返回的匿名函数和匿名函数中所使用到的变量所构成的整体叫做闭包。

new函数

用new去创建一个变量需要指定类型，并且创建的出来的是这个类型的指针。new 一般是给普通数据类型分配内存，后面还有一个make，是给引用数据类型分配内存的如channel、map、slice。

异常处理

Go语言追求简洁，所以不支持Java中那种的 try catch finally 这样的语法。

取而代之的是 defer、panic、recover来处理异常。recover是一个内置函数可以捕获到异常，一般在方法中使用defer 定义一个闭包里面执行recover去捕获异常。

上述代码执行结束虽然出现了异常，但是不会因此而停止运行后面的代码。但是有些时候一场可能是致命的我们希望出现异常既能够捕获又能够停止程序，那就需要用到errors包。

  • errors.New(“错误说明”) 可以自定义错误的信息，并且放回一个error类型的值，然后使用panic内置函数 可以将error类型的值传递给painc函数，就会打印出错误信息，并且结束程序。

用开头字母的大小写区别 public 和 private。

继承

go中的继承是在结构体中嵌入另一个结构体的名称，称为匿名结构体，嵌入了就可以使用对应的属性和方法。包括私有的成员变量。

  • 方法重写

多匿名结构体嵌入时，如果有重名字段需要指明操作哪个匿名结构体的属性。

接口

接口是一个引用数据类型。go中没有implement关键字，没有显示的声明接口的实现类，只要重写了接口中的所有方法就是这个接口的实现类。

一些标识

  • 可以在结构体中快速生成实现方法

  • go的接口设计不关心实现哪个接口，只关心实现了哪些方法，如果两个接口的方法完全一样，另一个类实现了所有方法，就同时实现了两个接口。
  • 接口是可继承的，要实现一个接口需要实现所有的方法，包括父类的方法。子接口不能继承有两个完全相同的方法的父接口
  • 引用类型实现和值类型实现是不一样的。

单元测试

之前说过 go的文件不能有下划线，这样说也不全对，因为下划线命名的文件会被当做是一个测试文件。

单元测试可以不需要main函数，就可以去测试其他的方法，但是有以下几点要求：

  • 文件必须以 _test.go 为结尾

  • 单元测试中的测试函数的名字必须是 TestXxxx的格式，要以Test开头并且下一个字母必须是大写。

  • 测试函数的方法需要接收一个 *t testing.T 参数

  • 待测试函数

  • 测试函数

原理就是，在测试文件中的所有满足上面的要求的函数都会被testing引入，它里面有一个隐藏的main函数，会帮我们调用所有的TestXxxx函数。

也可以单个函数进行测试

协程(goroutine)、管道(chanel)

在go中，一个线程可以启动多个协程，协程就是一个轻量级的线程。 协程必须在主线程执行结束之前结束，否则主线程结束会让协程也一起结束。

协程比线程更低一级的运行方式，它不属于操作系统功能，不由操作系统分配cpu、内存，而是由golang自己实现运行资源的调度使用。也正是因此执行效率要高于线程。

相比于Java这样的多线程并发编程语言，go采用的协程相比于线程占有的资源较少，上下文切换花费时间较少，速度更快。

goland 中用 go关键字去开启一个go协程。

执行上面的程序会发现，开启协程的那个方法根本没有执行。为什么？

这个需要结合go语言并发的模型，MPG模型

  • M（Machine）：操作系统的主线程

  • P（Processor）：协程执行需要的资源（上下文context），可以看作一个局部的调度器，使go代码在一个线程上跑，他是实现从N：1到N：M映射的关键

  • G（Gorountine）：协程，有自己的栈。包含指令指针（instruction pointer）和其它信息（正在等待的channel等等），用于调度。一个P下面可以有多个G

一开始运行go就会创建一个主协程(线程)，执行到go语句时候，就会创建一个调度点，这个协程就会挂在P的队列下面。当主协程被阻塞或者等待时，就会在P的队列下面取出一个协程进行执行。

这样也就很好解释为什么上面的代码 sayhello方法为什么没有被执行了，因为主协程是一直执行的，没有被阻塞，所以没有调度go 开启的协程，又因为协程必须在主协程执行结束前调度，所以 sayhello不会被调度。

如果在主协程上加上一个 sleep就能够调度到 sayhello函数。

这些都是需要解决的问题，因此下面引出了管道这么一个概念。

管道

管道的本质就是一个队列。并且它本身是线程安全的，channel使用时可以指定类型。chan是一个引用类型。满足FIFO的规则

  • 声明格式

var 变量名 chan 变量类型

注意：chan指定为 inferface{}的类型时，取出的都是这个类型，如果存入的是结构体，取出时需要类型转换。

  • chan 关闭

可以通过 close函数关闭一个chan，但是关闭后的管道仍然可以读但是不能写。close(channel)

  • 管道遍历

遍历时不能使用普通for 循环，要用foreach，并且遍历前需要将管道关闭，因为关闭才会有一个停止标记位。否则将出现死锁。

使用管道就能够解决我们上面的问题，加锁太慢，管道天然线程安全，不知道休眠时间，可以通过设定一个标志管道，主线程循环等待，当标志管道满足条件时，才退出。

  • chan可以声明为只读和只写，一般定义成双向管道，方法接收参数时候声明为其中一种，以防误操作。

  • 当不知道什么时候关闭管道，但是又要从里面取数据时，可以用select关键字防止死锁。select和switch一样会依次向下匹配知道找到满足条件的，否则执行default。
