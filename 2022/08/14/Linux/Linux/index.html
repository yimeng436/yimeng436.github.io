 Linux 

简单指令

  • cd

  • pwd

  • ls

    • ls -l 详细信息
    • ls -all 所有文件 包括隐藏文件
      • 里面列出来的文件 第一个字母 d ：表示文件夹 l：表示link文件 b：接口设备信息 - ：表示文件
      • 后面 rwx 分别表示读写可执行，三个为一组一个三组 ： 属主、属组、其他用户的权限，- 表示没有权限
      • 在切换权限时候 r=4 w=2 x=1

  • rm

  • mkdirt

    • -p 递归创建

  • mv [源地址] [目的地址] 移动

  • cp [源地址] [目的地址] 拷贝

  • chmod xxx filename 切换当前操作者对当前文件的权限

    • xxx 表示三个身份的权限值，7 = 4+2+1可读可写可执行 ，其他一样

  • cat 从第一行开始查看文件内容

  • tac 从最后一行开始查看文件内容

  • nl 代开的文件并且带有行号

  • more 打开文件不显示全部，空格代表翻页，回车代表一行

  • touch 创建文件

  • ln file1 file2 创建文件1的连接

    • 默认是硬连接，类似于复制了一个file1的文件并且是同步操作的
    • ln -s xx xx 表示创建 软连接 软连接类似于快捷方式，删掉源文件便不可访问

vim

Vim 就是一个功能很强大的文编编辑器，分为三种模式：命令模式、输入模式、底线命令模式

用户输入 Vim filename 就会进入命令模式：

此时还不能输入字符串，但是可以输入命令：i 开始插入内容， ： 进入底线命令模式 ， x： 删除当前光标所在的字符

  • **输入 i： ** 此时可以进行输入内容，如果要退出就通过 ESC键，退出后进入 命令模式

  • **输入：进入底线命令模式 ** 光标移动在最下面，这时候可以使用一些底线命令，w：保存文件 q：退出程序 wq 保存并退出

  • 其他一些功能：
    • 数字 + 空格 ：光标跳几个字符
    • 数字 + 回车： 光标跳几行
    • / +字符 查找 小写n代表 下一个 大写N代表上一个
    • set nu 显示行号

账号管理

用户管理

一般我们使用的时候拿到的不是root用户，账号管理只有root用户可以完成

添加一个用户 ：useradd -m username，就会创建一个用户，-m表示自动在 /root下创建对应的信息，所有配置信息都存在 etc目录下，用户的信息也会存放在etc下，在etc/password中可以看到所有创建了的用户的登录密码，文件位置等信息

useradd -m zyh root下会出现一个 zyh文件

删除用户： userdel -r username，加上 -r 表示会一并把所有配置文件删除包裹 /root下的和 etc中的

修改用户：

usermod：他有一些参数

  • -c 修改描述信息
  • -d 修改主目录路径
  • -g 修改用户所属的组
  • -G 分配用户组

这些参数和useradd 相同，

usermod -d /hone/test zyh 把原本的home/zyh的内容改到了/home/test中，但是这里要注意的是/home/zyh 仍然存在，并且 如果 home/test 不存在就不会被创建，但是etc/passwd 中 zyh这个用户的信息会被修改，主要还是要看配置文件。

切换用户

Linux中 只有当前为超级用户时，命令行以 #开头，其他都是$

su username 切换到username这个用户下

修改密码

  • 超级用户

passwd username 修改username这个用户的密码

  • 普通用户

passwd 修改当前用户的密码

在linux中输入密码时看不见的

用户组管理

每一个用户都可以属于一个用户组，系统对不同的组进行不同的管理，这些管理操作实际上都是对etc下面的配置文件进行管理，用户的时 etc/passwd，用户组的时 etc/group

添加用户组

groupadd groupname

创建一个组，并且通过id唯一标识，不指定就从1000开始自增

可以通过参数 -g指定

删除 groupdel

修改用户组 groupmod

  • -g 修改组id
  • -n 修改名字

切换用户组

newgrp groupname

进程管理

  • 查看当前系统正在执行的进程信息：

    ps -xxx

    • -a ：终端所有运行的进程
    • -u ：以用户信息显示进程
    • -x：后台进程运行的参数
    • -ef ：查看父进程的信息（一般使用结构树去查看）
      • pstree -pu，p表示显示父进程id，u表示显示用户组

  • 查看文件中符合条件的字符串

    • grep name

  • | 管道符，A|B，表示A通过B条件取过滤

​

把上面的几个命令连起来使用：

  • ps -aux|grep mysql 前面的表示查询所有的进程信息之后，通过管道符过滤，过滤的条件是所有和mysql相关的进程

  • 结束进程： kill -9 进程id
